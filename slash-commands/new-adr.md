Criar um novo ADR usando `templates/adr-template.md` como base e salvar em `docs/decisions/adr-XXXX-titulo.md` (XXXX sequencial, titulo em slug). Preencha obrigatoriamente:
- Contexto
- Problema
- Alternativas consideradas (comparando prós/contras)
- Decisão tomada (com status Proposed/Accepted/Rejected)
- Consequências (positivas, negativas, riscos)
- Notas de implementação (owners, rollout, flags)
- Referências (links para docs relacionados)

Cite stakeholders e conecte o ADR às demais seções de arquitetura afetadas.
